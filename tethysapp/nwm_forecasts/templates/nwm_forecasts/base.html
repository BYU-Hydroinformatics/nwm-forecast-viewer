National Water Model Forecast Viewer Guide

    About the App

    The NWM Viewer app has 2 modes:

    1. Home: Retrieve and view time series for a single stream reach, reservoir or grid cell (Home mode)
    2. Subset: Subset NWM outputs (NetCDF files) with a watershed polygon to get 'shrunken' NetCDFs that only contain data for a specific area.

    Home mode (switch to Subset mode for the other part)
  1. Add Watershed (optional) - Add a watershed polygon from HydroShare or local files (geojson or shapefile)
  2. Model Configuration
    NWM has 4 model configurations:
    1. Analysis & Assimilation runs hourly and provides a snapshot of current hydrologic conditions at each run
    2. Short Range runs hourly and provides a 18-hour (1-hour interval) deterministic forecast at each run
    3. Medium Range runs 4 times per day (every 6 hours) and provides a 10-day (3-hour interval) deterministic forecast at each run
    4. Long Range runs 4 times per day (every 6 hours) and provides a 4-member 30-day (6-hour interval) ensemble forecast at each run (16-member ensemble daily)
  3. Model Output File
    NWM stores model outputs in different NetCDF files.
    1. Channel file: stream routing result for 2.7 millions stream reaches (point-type NetCDF)
    2. Land file: forecasts land surface processing result for continental US (1km gridded NetCDF)
    3. Reservoir file: forecasts for 1260 reservoirs (point-type NetCDF)
    4. Forcing file: forcing variables for NWM (1km gridded NetCDF)
  4. Variable: choose a variable (available variables may vary depending on previous Model Configuration and Model Output File)
  5. COMID / Grid Cell Index:
    1. COMID: the unique and permanent identifier assigned to each stream reach and reservoir in NHD+ system. You can either manually enter a COMID or select a stream reach or reservoir on map
    2. Grid Cell Index: a pair of index value assigned to each grid cell. You can either manually enter the index values or select a grid cell on map
  6. Date and Time
    The date and time options vary depending on Model Configuration:
    1. Begin/End Date: a string YYYY-MM-DD to indicate model run date or date range
    2. Model Initialization Time: time point at which model starts to run. Short Range models runs hourly (24 runs per day); Medium Range model runs every 6 hours (4 runs per day).
    3. Ensemble Model Initialization Time: time point at which model starts to run. Long Range model runs every 6 hours (4 runs per day). Each run produces a 4-member 30-day forecast.
  7. View Forecast - Click 'View Forecast' button to retrieve time series

    About Subsetting

    Users can subset NWM outputs (NetCDF files) by a watershed polygon to get 'shrunken' NetCDFs that only contain data for this specific area.

      Long Range: {{date_string_oldest}} to {{date_string_today}}

    How to Subset a Watershed
  1. Add Watershed - Add a watershed polygon from HydroShare or local files
  2. *Configuration - Select a forecast range: Short, Medium, Long, or Analyis & Assimilation
  3. *Geometry - Select a geometry: Channel, Land, Reservoir or Forcing
  4. Date and Time - Select a date: The date and time options will vary depending on the forecast type
  5. Upload to HydroShare (optional) - Check 'Upload to HydroShare' to create a HydroShare resource for subsetted netcdfs
  6. Download subsetting results - Click 'Download' button
